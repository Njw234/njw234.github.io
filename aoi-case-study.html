UX Case Study

Agents of Influence

A UI/UX case study focused on improving onboarding clarity, fixing the student password reset flow, and streamlining the teacher dashboard for an educational misinformation awareness game.

Role UI/UX Designer
Tools Figma, User Flow Mapping, Prototyping
Team Small Indie Studio
View Figma See Process

Project Overview

The platform needed clearer onboarding for teachers, a working password recovery path for students, and a less overwhelming dashboard. The goal was to reduce cognitive overload and streamline key user journeys.

Figma Prototyping User Flows
01

The Problems

After auditing the platform, three distinct friction areas were identified across the teacher and student experience.

🔐

Broken Password Reset Flow

Students had no clear path to recover their account. After three failed login attempts the system offered no guided recovery — leaving students locked out and dependent on their teacher to intervene manually.

📋

Overloaded Teacher Dashboard

The lesson management dashboard presented too many actions simultaneously. Teachers struggled to identify their primary tasks, and the visual hierarchy gave equal weight to unrelated controls, creating decision fatigue.

🚪

Unclear Onboarding Entry Points

New teachers had no guided first-run experience. The initial login dropped users directly into the dashboard with no orientation, causing confusion about where to start and how to set up their first classroom.

02

Process

1

Flow Audit

Mapped existing onboarding and password reset flows end-to-end. Flagged every dead end, redundant step, and unclear label across both the student and teacher experiences.

2

Wireframing

Created low-fidelity layouts for each problem area. Focused on progressive disclosure — only surfacing what the user needs at each step rather than all options at once.

3

High-Fidelity Prototyping

Built interactive Figma prototypes for the revised password reset flow, teacher onboarding, and dashboard hierarchy. Each flow was tested independently before being combined.

Results & Impact

Password Recovery

Students now have a clear, guided path to recover their account without teacher intervention, reducing classroom disruptions.

Onboarding Clarity

Reduced initial confusion by surfacing only the most critical actions on first login, guided by progressive disclosure principles.

Dashboard Usability

Streamlined the teacher dashboard — primary actions are now visually prioritised and secondary controls are grouped and deprioritised.

Educator Experience

Contributed to a more polished platform that better supports teachers running the game in a classroom setting.
